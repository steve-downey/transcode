## What was done

- Created `tools/process_benchmark_results.py`:
  - Parses one or more Catch2 XML files produced via `--reporter xml`
  - Loads corpus byte counts from `data/benchmarks/corpus_manifest.json` (downloaded)
    and/or `benchmark/corpus/` (checked-in small files)
  - Infers the corpus file for each benchmark via a priority-ordered hint table
    (`CORPUS_NAME_HINTS`); more-specific patterns (windows-1251, Shift-JIS) win
    over generic keywords (English, Arabic, Russian)
  - Computes throughput in MiB/s: `corpus_bytes / (mean_ns × 10⁻⁹) / 2²⁰`
  - Renders a Markdown table (name, corpus bytes, mean µs, ±CI, MiB/s)
  - Optionally writes a Vega-Lite JSON bar-chart spec (`--vegalite PATH`)
  - Return type of `render_vegalite_spec` is `dict[str, Any]` (avoids double-subscript
    mypy errors on `dict[str, object]`)
- Created `tools/tests/test_process_benchmark_results.py`: 40 pytest tests covering
  all public functions
- Created `docs/benchmarks/README.md`: full benchmark guide (corpus files, quick
  start, throughput formula, report commands, matrix reproduction steps)
- Modified `Makefile`:
  - `bench-report`: build RelWithDebInfo, run smoke, save XML, render Markdown report
  - `bench-lto-xml`: configure (if needed), build LTO benchmarks, save XML
  - `bench-report-lto`: run bench-lto-xml then process_benchmark_results with Vega-Lite

## Implementation notes

- `bench-report` always overrides `CONFIG=RelWithDebInfo` so it never runs with
  Asan overhead; the recipe uses the Multi-Config `.build/build-system/` tree.
- `bench-lto-xml` is a separate helper so `bench-report-lto` can be composed
  cleanly without duplicating the LTO build logic.
- The corpus manifest `data/benchmarks/corpus_manifest.json` is NOT committed
  (generated by `uv run python tools/download_benchmark_corpora.py`). The tool
  falls back to the small checked-in corpus in `benchmark/corpus/` when the
  manifest is absent.
- "Japanese" is deliberately absent from `CORPUS_NAME_HINTS` because it is
  ambiguous between `ja_mars_utf8.txt` and `ja_mars_shiftjis.bin`; benchmarks
  named "Shift-JIS Japanese" match via the more-specific "Shift-JIS" hint.
