# Benchmark Corpora — Wikipedia Mars Article

Pristine-ish corpus data. **Do not edit manually.**
To refresh: `uv run tools/download_benchmark_corpora.py`

## Purpose

Real-world text corpora for Unicode transcoding performance benchmarks.
Each corpus targets a specific encoding stress pattern:

| Language | Encoding stress | Files |
| -------- | --------------- | ----- |
| English  | ASCII-heavy UTF-8 | `en_mars_utf8.txt` |
| Arabic   | Mixed 1/2-byte UTF-8 | `ar_mars_utf8.txt` |
| Russian  | Cyrillic UTF-8 / windows-1251 | `ru_mars_utf8.txt`, `ru_mars_windows1251.bin` |
| Japanese | CJK UTF-8 / Shift-JIS | `ja_mars_utf8.txt`, `ja_mars_shiftjis.bin` |

## Encoding Conversions

The binary files (`*.bin`) are derived from the UTF-8 sources by Python's
standard codec:

```python
text.encode('windows-1251', errors='replace')   # -> ru_mars_windows1251.bin
text.encode('shift_jis', errors='replace')       # -> ja_mars_shiftjis.bin
```

Characters outside the target encoding are replaced with `?` (0x3F).

## Fallback Corpus

`benchmark/corpus/` contains short (~100-500 byte) hand-written representative
samples checked into the repository. These allow `make bench` to work without
network access.

The full corpora (hundreds of KB each) live in `data/benchmarks/` (gitignored)
and are populated by running `uv run tools/download_benchmark_corpora.py`.

## Manifest

`data/benchmarks/corpus_manifest.json` is generated by the download tool and
records file, language, encoding, byte count, and SHA-256 for each corpus file.
